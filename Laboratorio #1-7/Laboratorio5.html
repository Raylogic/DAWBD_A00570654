1) ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor? (1)

  • Para mantener la integridad y consistencia de los datos, así como evitar errores de usuario. Se recomienda que el sistema le especifique al usuario cómo espera recibir la información.

2) ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript? (1)

  • Usando atributos de validación proporcionados por HTML5 (Ej. required, invalid, minlength, maxlength, pattern, etc...).

3) Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica? (1)

  • Javascript ofrece más herramientas de validación que HTML5
  • Javascript permite generar mensajes de error personalizados
  • Los atributos de validación de HTML5 no funcionan en todos los navegadores
