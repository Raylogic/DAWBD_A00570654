1) ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor? (1)

  • Para mantener la integridad y consistencia de los datos, así como evitar errores de usuario. Se recomienda que el sistema le especifique al usuario cómo espera recibir la información.

2) ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

  • Inspecciona el HTML y desactiva las funcionalidades de Javascript manualmente

3) Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

  • Para poder validar los datos que el usuario ingresa con la base de datos conectada
