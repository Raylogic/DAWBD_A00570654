Preguntas:

1) ¿Por qué es una buena práctica separar el modelo del controlador? (1)

  • Permite reutilizar código para futuros programas
  • Flexibiliza la forma de desplegar la información
  • Facilita el manejo de memoria al desusar vistas fuera de pantalla

2) ¿Qué es SQL injection y cómo se puede prevenir? (2)

  • Vulnerabilidad que permite inyectar código SQL para interferir con los queries de una base de datos con el fin de extraer/modificar datos.
  • Puede provocar desde cambios persistentes en una aplicación web o creación de puertas traseras, hasta comprometer el funcionamiento de un servidor e incluso dejar obsoleta una base de datos.

¿Cómo prevenirlo?

  1. Queries parametrizados = Permite pre-compilar una sentencia SQL para identificar qué variables deben insertarse en ella para su ejecución
  2. Stored Procedures = Hace que la aplicación trate los inputs como datos en vez de comandos SQL
  3. Validación de inputs = Asegura que los inputs cumplan con los formatos especificados
  4. Limitar privilegios = Evita conectarse al núcleo de la base de datos con una cuenta
  5. Mensajes de error genéricos = Evita mostrar información sobre la arquitectura de la base de datos
